shipmine¶

Installation¶

(Optional) Create a conda environment:

conda create --name shipmine_env python=3.13
conda activate shipmine_env

You can install this package using pip:

pip install -e .

And install dependencies from requirements.txt:

pip install -r requirements.txt

Usage¶

Here is a basic example of how to use this project:

from shipmine import some_function
from shipmine.module.module_core import some_module_function
result = some_function()
print(result)

Description of each folder:¶

  • src/shipmine/:

    • core.py: Example file where library functions and logic are implemented. Rename this file with a more descriptive name for your specific project.

  • /tests/: Stores unit and integration tests to ensure the project functions as expected. Use pytest to run these tests effectively.

    • /tests/data/: If your unit tests need specific data, use this folder to store it.

  • /docs/: Includes project documentation, such as DDA and Standards. For large files, refer to their location on OneDrive instead of uploading them directly to git.

    • /images/: Folder to store images displayed in the readme.

  • /scripts/: Reserved section for auxiliary scripts that assist with project maintenance and deployment. This does not include the core logic of the application but rather tasks to facilitate its operation.

  • /examples/: Presents samples and examples of how to use the code in different scenarios. Useful for users who want to quickly understand the project’s capabilities or learn good implementation practices.

    • examples/main_example.py: Recommended script as a starting point for using the library. This file shows how to import and use the functions from core.py.

  • /notebooks/: Stores Jupyter Notebooks for interactive analyses and demonstrations. It’s crucial that these notebooks are clear and contain detailed comments, allowing other developers and users to understand visualizations, methodologies, and results efficiently. They should explain not only the “how” but also the “why” behind the presented results.

  • /data/: Here you will store data for tests, examples, etc. For more information on managing large files, see Data.

Running Tests¶

  • To run all tests within the project, execute:

    pytest

  • To run tests for a funciton, such as add_numbers in core.py you should execute TestAddNumbers in test_core.py (Check 3. Parallel Code and Test Structure in Style Guide):

    pytest tests/test_core.py::TestAddNumbers

Tox¶

To run all the tests and code checks, simply run:

tox

Tox expected output¶

After execting tox and everithing passes correctly you should get a output similar to:

Wrote HTML report to test_reports/htmlcov\index.html
  format: OK (1.10=setup[0.14]+cmd[0.96] seconds)
  lint: OK (1.06=setup[0.03]+cmd[1.03] seconds)
  py: OK (14.56=setup[13.04]+cmd[1.52] seconds)
  report: OK (1.07=setup[0.03]+cmd[0.58,0.45] seconds)
  congratulations :) (17.94 seconds)

  • If the lint phase fails you can try to fix it automatically with ruff check --fix. If there are error not able to be fixed automatically you should fix them manually.

  • If the py (Testing) phase fails you should revise your code and/or your test to fix them.

Version management¶

This template provides a basic bumpversion configuration. This tool help to change all the places where version is stored at the same time. It’s as simple as running:

  • bumpversion patch to increase version from 1.0.0 to 1.0.1.

  • bumpversion minor to increase version from 1.0.0 to 1.1.0.

  • bumpversion major to increase version from 1.0.0 to 2.0.0.

You can use bumpversion patch --commit --tag to create a commit and a tag with the version name automatically.

Sphinx¶

To generate the documentation with sphinix you can execute the following commands:

sphinx-build -b doctest docs dist/docs
sphinx-build -b html docs dist/docs
sphinx-build -b linkcheck docs dist/docs

The html documentation should have been generated in dist/docs. Open index.html in your web browser to navigate through it.

Contributing¶

If you wish to contribute to this project, please follow these steps:

  1. Read the style guide

  2. Clone the repository.

  3. Create a branch for your changes (git checkout -b feature/new-functionality).

  4. Run code style tests, unit tests and coverage test by executing tox and ensure all tests pass correctly.

  5. Commit your changes (git commit -m 'Adding new functionality').

  6. Push to the branch (git push origin feature/new-functionality).

  7. Submit a Pull Request.
